# Anatomy

For the works published here, the FRBR conceptual model is used to describe the different levels of abstraction[^1].

![](/assets/images/cm-project.drawio.svg){: .d-block .mx-auto }

Works and the versions which express it are abstract. Each such version can then be embodied in one or more
(physical) manifestations which can be interacted with.

{: .note-title }
> Note
> 
> A _work version_ is our specialization of a FRBR expression which denotes the version of a work.

<details open markdown="block">
    <summary>Example: Capaciteitskaart</summary>
    {: .text-delta }
![](/assets/images/cm-capaciteitskaart.drawio.svg){: .d-block .mx-auto }
</details>


Currently, the following types of works are distinguished:

Types of work
{: .text-delta }
<dl>
  <dt>Information model</dt><dd>Specification of semantics and constraints of the use case represented by the data product, including documentation and supporting artifacts.</dd>
  <dt>Documentation</dt><dd>Documentation which enables or supports the (data) modeling of data products.</dd>
</dl>

{: .note-title }
> Beware of colloquial speech
>
> Throughout this documentation the term _(information) model_ may be used more colloquially.
>
> For example, I'm unlikely to say I have openend in my text editor an exemplar of the LinkML manifestation of the version 1.2 expression of the work Capaciteitskaart, and may instead refer to it as "Capaciteitskaart model 1.2", or in some contexts even as "the Capaciteitskaart model".
